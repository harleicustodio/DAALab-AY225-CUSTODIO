This project is a browser-based TSP Route Network Analyzer that finds the shortest path between eight cities in Cavite — Bacoor, Dasma, Imus, Noveleta, Kawit, Silang, Gentri, and Indang, using Dijkstra's algorithm. Built with plain HTML, CSS, and JavaScript, the tool lets users select a start and end node and choose an optimization metric (distance, time, or fuel). Dijkstra's algorithm works by assigning a cost of zero to the start node and infinity to all others, then repeatedly relaxing edges from the lowest-cost unvisited node until the destination is reached. The resulting path is reconstructed by tracing backwards through a previous-node table and is displayed simultaneously on an animated SVG map and a synchronized edge data table.
The biggest challenge was implementing the algorithm correctly from scratch, early versions produced wrong paths because the path reconstruction step was not traced back properly, which took multiple rounds of debugging to fix. Positioning nodes on the SVG canvas manually and computing arrowhead directions using unit vectors was also tricky, as was styling the three-column CSS Grid layout so it held together across screen sizes. Keeping the map and table in sync, especially clearing leftover highlights between searches, required careful state management. For references, I looked at classmates' HTML and CSS layouts for design inspiration, and used Claude (Anthropic's AI) to clarify the path reconstruction logic and troubleshoot CSS Grid column sizing. All code and design decisions were ultimately my own.
